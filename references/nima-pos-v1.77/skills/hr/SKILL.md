# نظام الموارد البشرية والرواتب (HR & Payroll System)

هذا الملف (Skill) يوفر معايير هندسية متكاملة لبناء وتوسيع نظام الموارد البشرية والرواتب داخل التطبيق، معتمداً على بيئة العمل الحالية (React + Dexie.js) لضمان العمل بدون اتصال بالإنترنت (Offline-First).

أنت وكيل برمجي محترف. المطلوب بناء نظام موارد بشرية ورواتب (HRMS) متكامل وحديث باللغة العربية RTL، مصمم ليدير الموظفين، الحضور والانصراف، الهيكل التنظيمي، وحساب الرواتب المعقد باحترافية وسهولة، مع الترابط التام مع الـ ERP أو النظام المحاسبي.

## أولاً: أقسام وتطبيقات النظام الشاملة (Comprehensive Modules)
يوفر هذا النظام وحدات الموارد البشرية التالية:
1. **لوحة الـ HR (HR Dashboard)**: ملخص أعداد الموظفين والإجازات.
2. **ملفات الموظفين (Employee Profiles)**: العقود والمرفقات وبيانات العاملين.
3. **الحضور والانصراف (Time & Attendance)**: مراقبة التأخيرات والغياب.
4. **الإجازات والسلف (Leaves & Loans)**: طلبات وتتبع أرصدة الإجازات.
5. **مسير الرواتب (Payroll Run)**: توليد الرواتب آلياً بالاستقطاعات والمكافآت.
6. **نهاية الخدمة والتقييم (End of Service)**: التقييم وإجراء المخالصات للمستقيلين.

## ثانياً: المعايير التقنية والتصميمية (Core Guidelines)

### 1. الأثر المالي والقيود المحاسبية (Global Financial Impact) - **مهم جداً**
- **مسير الرواتب (Payroll)**: عند اعتماد الرواتب نهاية الشهر، ينشئ النظام قيداً محاسبياً مجمعاً (مصروف رواتب / ذمم رواتب مستحقة) ولا توزع كنقدية إلا عند الدفع.
- **السلف والعهد (Loans)**: تسجل كذمم للموظفين (أصول)، وتُستقطع دورياً في مسير الرواتب بنظام السداد.
- **مكافأة نهاية الخدمة (Indemnity)**: التزامات ومصروفات تُسجل وتُرحل آلياً مع المخالصة.

### 2. هيكل قاعدة البيانات المقترح لـ HR (Dexie Models)
```typescript
// بيانات الموظف
export interface Employee {
  id?: number;
  code: string; // كود وظيفي فريد
  firstName: string;
  lastName: string;
  nationalId: string;
  departmentId: number;
  positionId: number;
  joiningDate: string;
  birthDate: string;
  gender: 'Male' | 'Female';
  maritalStatus: 'Single' | 'Married' | 'Divorced' | 'Widowed';
  contact: { phone: string; email?: string; address: string; emergencyPhone?: string };
  bankDetails: { bankName?: string; iban?: string; accountName?: string };
  status: 'Active' | 'OnLeave' | 'Terminated' | 'Resigned';
  isActive: boolean;
}

// العقد والبيانات المالية
export interface EmployeeContract {
  id?: number;
  employeeId: number;
  contractType: 'Full-time' | 'Part-time' | 'Temporary' | 'Contractor';
  startDate: string;
  endDate?: string; // أتركه فارغ إذا كان العقد غير محدد المدة
  basicSalary: number;
  housingAllowance: number;
  transportAllowance: number;
  otherAllowances: number;
  socialInsuranceNumber?: string;
  insuranceDeduction: number;
  taxDeduction: number;
}

// مسير الرواتب (Payroll Run)
export interface PayrollRun {
  id?: number;
  period: string; // مثلاً 2026-05
  status: 'Draft' | 'Approved' | 'Paid';
  totalBasicSalary: number;
  totalAllowances: number;
  totalDeductions: number;
  totalNetPay: number;
  postedJournalEntryId?: number;
  createdBy?: number;
}

// قسيمة الراتب لكل موظف (Payslip)
export interface Payslip {
  id?: number;
  payrollRunId: number;
  employeeId: number;
  basicSalary: number;
  allowances: any; // تفاصيل البدلات
  deductions: any; // تفاصيل الاستقطاعات (غياب، سلف، جزاءات، ضرائب)
  netPay: number;
  paymentStatus: 'Unpaid' | 'Paid';
}
```

## ثانياً: المتطلبات الوظيفية المفصلة للنظام (Functional Requirements)

========================
1. لوحة القيادة للموارد البشرية (HR Dashboard)
========================
- إجمالي عدد الموظفين النشطين حسب الإدارة/القسم.
- الحضور والغياب اليومي (أرقام سريعة).
- الموظفون في إجازة حالياً.
- الإقامات، العقود، أو الجوازات التي أوشكت على الانتهاء.
- إجمالي الرواتب للشهر الحالي مقابل الشهر السابق.
- تنبيهات أعياد ميلاد الموظفين.

========================
2. الهيكل التنظيمي (Organizational Structure)
========================
- إعداد الشركات أو الفروع (إن وجدت).
- إعداد الإدارات (المالية، الموارد البشرية، المبيعات... إلخ).
- إعداد المسميات الوظيفية (Positions/Titles) وربط كل مسمى بالمهام والوصف الوظيفي.
- عرض شجري مرئي (Tree View) أو هرمي للهيكل التنظيمي للمؤسسة.

========================
3. إدارة الموظفين (Employee Profile & Onboarding)
========================
- ملف موظف (Profile) شامل مقسم إلى Tabs:
  - **البيانات الشخصية**: الاسم، الهوية، تاريخ الميلاد، الحالة الاجتماعية، الجنسية.
  - **بيانات العمل**: الإدارة، المسمى الوظيفي، مدير مباشر، تاريخ التعيين، نوع الدوام.
  - **البيانات المالية والعقد**: الراتب الأساسي، البدلات بأنواعها، طريقة الدفع (كاش/بنك)، رقم الحساب (IBAN).
  - **الوثائق والمرفقات**: صورة الهوية، جواز السفر، السيرة الذاتية، الشهادات، العقد الموقّع. (مع تنبيهات بتاريخ انتهاء الصلاحية).
  - **السجل الوظيفي (Job History)**: الترقيات، نقل الإدارات، تعديلات الراتب.

========================
4. نظام الحضور والانصراف (Time & Attendance)
========================
- تعريف ورديات العمل (Shifts) وأوقات الدوام الافتراضية، وفترات السماح (Grace Period) للتأخير.
- إدخال الحضور اليدوي أو عن طريق استيراد ملف Excel/CSV من ماكينات البصمة.
- تسجيل الإضافي (Overtime) بأنواعه (دوام عادي، يلة، إجازة رسمية).
- شاشة مراقبة اليومية لمعرفة الغائبين والمتأخرين فوراً.
- حساب ساعات الغياب والتأخير وتحويلها إلى استقطاعات بشكل آلي نهاية الشهر.

========================
5. إدارة الإجازات والمغادرات (Leave & Permission Management)
========================
- إعداد أرصدة الإجازات السنوية (مثلاً 21 أو 30 يوم حسب قانون العمل).
- أنواع الإجازات: سنوية، مرضية، عارضة، أمومة، بدون راتب، حج.
- نظام طلبات الإجازات (Leave Requests): يقدم الموظف طلباً أو يُدخله الـ HR، ويحتسب أيام العمل الصافية بمعزل عن العطلات الرسمية والأسبوعية.
- خصم الإجازة تلقائياً من رصيد الموظف.
- المغادرات الساعية (أذونات الخروج) لمعرفة الوقت المستهلك من قبل الموظف أثناء الدوام.

========================
6. السلف والقروض (Loans & Advances)
========================
- طلب سلفة نقدية.
- جدول سداد السلفة (شهرياً).
- الخصم الآلي لقسط السلفة في مسير الرواتب نهاية كل شهر.
- منع الموظف من أخذ سلفة تتجاوز راتبه (حسب سياسة الشركة) أو سلفة جديدة قبل سداد القديمة.

========================
7. الجزاءات والمكافآت (Penalties & Bonuses)
========================
- لائحة الجزاءات (تأخير، غياب، مخالفات إدارية) وتأثيرها (إنذار، خصم يوم، خصم نسبة).
- تسجيل المكافآت المالية والعمولات التي تدرج آلياً في مسير الرواتب القادم.

========================
8. مسير الرواتب والأجور (Payroll Generation)
========================
هذه هي النواة الأساسية للنظام ولها قدر كبير من الأهمية:
- **تحضير المسير (Run Payroll)**: اختيار الشهر، يقوم النظام بجمع جميع الموظفين النشطين ويحسب:
  - الراتب الأساسي المستحق (بناء على عدد أيام العمل أو الغياب).
  - إضافة كافة البدلات (سكن، مواصلات، طعام).
  - إضافة الإضافي، المكافآت والعمولات.
  - خصم الغياب والتأخير غير المبرر.
  - خصم أقساط السلف المستحقة في هذا الشهر.
  - خصم حصة המوظف في التأمينات الاجتماعية والضرائب.
  - احتساب الصافي المستحق (Net Pay).
- **مراجعة المسير (Review & Edit)**: إمكانية تعديل رواتب أفراد بعينهم قبل الاعتماد النهائي (بإضافة خصومات منفردة أو غيرها).
- **الاعتماد والصرف (Post/Approve)**: بمجرد الاعتماد، يغلق المسير وتولد القيود المحاسبية لترحيل الرواتب المستحقة لإدارة الحسابات لصرفها.
- **توليد قسيمة الراتب (Payslip)**: طباعة وتصدير قسيمة الراتب للموظف. وإمكانية إرسالها عبر الواتساب أو الإيميل أو تطبيق الموظف.

========================
9. التأمينات والضرائب (Taxes & Social Insurances)
========================
- نظام ديناميكي لحساب ضرائب الدخل أو كسب العمل (Income Tax Brackets).
- إعداد نسب التأمينات الاجتماعية (كم حصة الشركة، وكم حصة الموظف).

========================
10. نهاية الخدمة والمخالصة (End of Service & Offboarding)
========================
- إنهاء عقد الموظف أو الاستقالة.
- حساب مكافأة نهاية الخدمة (Indemnity/Gratuity) بناءً على قوانين العمل المحلية (مثلاً أجر نصف شهر عن أول 5 سنوات، ثم شهر كامل).
- تصفية الرصيد المتبقي لإجازات الموظف وصرفه نقدياً (Leave Encashment).
- خصم أي سلف أو التزامات متبقية على الموظف في المخالصة النهائية.
- أرشيف الموظف بعد نهاية الخدمة (لا يتم حذف أوراقه لضمان السجلات القانونية).

========================
11. تقييم الأداء (Performance Appraisal)
========================
- نماذج تقييم سنوية، نصف سنوية، أو ربعية.
- معايير الأداء (KPIs) وربط التقارير بقرارات زيادة الراتب والمكافآت الدورية.

========================
12. التقارير الاستراتيجية والتشغيلية (HR Reports)
========================
- تقرير كشف الرواتب المفصل (Payroll Summary) للشهر.
- ملف تحويل الرواتب للبنك (WPS - Wage Protection System).
- تقرير إحصائي للحضور والغياب والإضافي.
- تقرير الحركات المالية للموظف، وبطاقة رصيد الإجازات.
- تقرير التأمينات والضرائب الشهري المستحق سداده للجهات الحكومية.

========================
13. بوابة الموظف (Self-Service / Employee Portal) (إضافة متقدمة)
========================
- واجهة خاصة للموظف يدخل إليها بحسابه:
  - يستعرض قسائم رواتبه.
  - يقدم طلبات إجازة وسلفة وينتظر موافقة المدير الـ HR.
  - يعرض أرصدة إجازاته واستقطاعاته.

========================
14. متطلبات واجهة المستخدم والميزات المتقدمة
========================
- **تصميم التقارير والمسير**: استخدم عروض جداول عرضية DataGrids قوية تتحمل العشرات من الأعمدة في شاشة الرواتب.
- التنبيهات المنبثقة للـ HR بالطلبات المعلقة (إجازة، سلفة).

========================
15. اختبارات القبول (Acceptance Criteria)
========================
بعد التنفيذ يجب التأكد من:
1. إضافة موظف جديد بعقد مالي مفصل.
2. تسجيل سلفة على الموظف وتحديد أقساطها.
3. معالجة دوام وغياب الموظف.
4. توليد مسير رواتب لجميع الموظفين ورؤية خصم قسط السلفة والغياب، وإضافة البدلات، والتحقق من حساب الصافي.
5. اعتماد المسير والتأكد من إرسال قيد يومية للحسابات العامة، وتحضير الرواتب للدفع.
6. إجراء تسوية متبقية ونهاية خدمة للموظف.
