# نظام حضانة ومدرسة متكامل

هذا الملف (Skill) يوفر معايير هندسية متكاملة لبناء وتوسيع نظام إدارة الحضانة والمدرسة داخل التطبيق، معتمداً على بيئة العمل الحالية (React + Dexie.js) لضمان العمل بدون اتصال بالإنترنت (Offline-First).

أنت وكيل برمجي محترف. المطلوب بناء نظام إدارة حضانة وروضة أطفال متكامل وحديث باللغة العربية RTL، مستوحى من نظام Kids Home القديم، لكن بتصميم حديث ومنظم وسهل الاستخدام.

الهدف:
إنشاء نظام كامل لإدارة حضانة/روضة يشمل الأطفال، أولياء الأمور، الفصول، الاشتراكات، الماليات، الحضور، الموظفين، المرتبات، التقييمات، الصحة، الباص، التقارير، والصلاحيات.

مهم جدًا:
لا تنشئ واجهات مزدحمة مثل النظام القديم. استخدم تصميم حديث، واضح، RTL، يدعم اللغة العربية بالكامل، ويقسم البيانات الكبيرة إلى Tabs وصفحات فرعية.

## أولاً: أقسام وتطبيقات النظام الشاملة (Comprehensive Modules)
يوفر النظام الوحدات التالية لتغطية عمليات الحضانات وحضانة الأطفال:
1. **لوحة الحضانة (Nursery Dashboard)**: ملخص أعداد الأطفال، الشواغر، المتأخرات النقدية.
2. **إدارة الأطفال والأسر (Students & Guardians)**: ملفات الأطفال وتقييماتهم والأشخاص المصرح لهم بالاستلام.
3. **الاشتراكات والرسوم (Subscriptions & Fees)**: رسوم الحضانة، الباص، والأنشطة الإضافية.
4. **الفصول والجداول (Classes & Schedules)**: تقسيم الأطفال لمستويات ومتابعة السعة.
5. **التقييمات والتقارير اليومية (Daily Reports)**: تقارير الغذاء، النوم، والسلوك.
6. **الصحية والعيادة (Health Records)**: متابعة التطعيمات والأدوية والحساسية.

## ثانياً: المعايير التقنية والتصميمية (Core Guidelines)

### 1. الأثر المالي والقيود المحاسبية (Global Financial Impact) - **مهم جداً**
- **رسوم الاشتراكات**: تعتبر إيرادات محققة دورياً، ويتم تسويتها من خلال فواتير دورية وسندات قبض من أولياء الأمور وتسجل آلياً كقيود في `journalEntries`.
- **أجور المشرفات والسائقين**: مصاريف وتكاليف مباشرة تُسجل كمصروفات رواتب بنهاية الشهر.
- **التكامل مع المخزون**: الأدوات التعليمية، الزي المدرسي، والمواد المستهلكة تُنقص من المخزون وتضاف لفواتير الطلاب إن كانت مباعة.

### 2. هيكل قاعدة البيانات المقترح للحضانة (Dexie Models)
عند توسيع قاعدة البيانات في `db.ts`، يجب أن تتبع النماذج شكل مقارب للآتي:

```typescript
// جدول الأطفال
export interface SchoolStudent {
  id?: number;
  code: string; 
  name: string;
  nationalId?: string;
  dob: string;
  enrollmentDate: string;
  levelId: number;
  classId: number;
  status: 'نشط' | 'متوقف' | 'منسحب' | 'متخرج';
  guardianId?: number; // ربط مع ولي الأمر
  notes?: string;
}

// جدول أولياء الأمور
export interface Guardian {
  id?: number;
  name: string;
  relation: string;
  phone: string;
  whatsapp?: string;
  nationalId?: string;
  job?: string;
}
```

### 5. الأمان والتشغيل السليم
- استرجاع بيانات المستخدم:
```javascript
const userStr = localStorage.getItem('nima_user');
const currentUser = userStr ? JSON.parse(userStr) : null;
```
- تسجيل العمليات في الـ Audit:
```javascript
import { logActivity } from '../../utils/logger';
await logActivity('students', 'إضافة', `تم إضافة طفل جديد ${name}`);
```

### 6. قائمة المهام عند التنفيذ (Checklist)
عند بناء مكونات المدرسة بتوجيه هذا الـ Skill يجب التأكد من:
1. تقسيم الكود لمكونات واضحة وتجنب الملفات الضخمة المعقدة.
2. دعم التعديل السلس للمرفقات.
3. معالجة وحفظ البيانات حصرياً من خلال `db.ts`.
4. التحقق من صلاحيات العرض والتعديل والتصدير.

- جدول أطفال واضح.

- بحث بالكود.

- فلترة حسب المستوى.
- فلترة حسب الفصل.
- فلترة حسب الحالة: نشط / متوقف / منسحب / متخرج.
- زر إضافة طفل.

- زر تعديل.
- زر أرشفة.

ملف الطفل يجب أن يكون Tabs:

6. المدفوعات
7. الحضور والغياب
8. التقييمات
9. الصحة والتغذية
10. الملاحظات
11. سجل العمليات

حقول البيانات الأساسية:
- كود الطفل تلقائي وفريد.
- اسم الطفل بالعربية.
- اسم الطفل بالإنجليزية.
- الرقم القومي.
- تاريخ الميلاد.
- السن محسوب تلقائيًا.
- تاريخ الالتحاق.
- المستوى.
- الفصل.
- النوع.
- الجنسية.
- الديانة.
- العنوان.
- صورة الطفل.
- الحالة: نشط / متوقف / منسحب / متخرج.
- ملاحظات عامة.

ضع Validation:
- الرقم القومي لا يتكرر.
- كود الطفل لا يتكرر.
- الاسم مطلوب.
- المستوى مطلوب.
- الفصل مطلوب.
- تاريخ الميلاد مطلوب.

========================
22. اختبارات القبول
========================

بعد التنفيذ يجب تقديم تقرير يحتوي على:
- الملفات التي تم إنشاؤها.
- الشاشات التي تم تنفيذها.
- الجداول التي تم إنشاؤها.
- الصلاحيات المنفذة.
- الاختبارات التي نجحت.
- ما لم يتم تنفيذه.
- المشاكل المعروفة.

اختبر السيناريوهات التالية:
1. إضافة طفل جديد.
2. منع تكرار الرقم القومي.
3. إضافة ولي أمر وربطه بطفلين.
4. إضافة فصل بسعة محددة.
5. منع إضافة طفل لفصل ممتلئ.
6. إنشاء فاتورة لطفل.
7. تسجيل دفعة.
8. طباعة إيصال.
9. إلغاء إيصال بصلاحية مدير فقط.
10. تسجيل حضور طفل.
11. تسجيل انصراف واختيار المستلم.
12. تسجيل غياب.
13. إضافة تقييم شهري.
14. إنشاء شهادة PDF.
15. إضافة موظف.
16. تسجيل غياب موظف.
17. حساب مرتب.
18. إضافة مصروف.
19. عرض تقرير المتأخرين في السداد.
20. إنشاء نسخة احتياطية.
21. استعادة نسخة احتياطية.
22. التأكد أن المعلمة لا ترى إلا أطفال فصلها.
23. التأكد أن المحاسب لا يعدل بيانات الطفل الصحية.
24. التأكد أن كل عملية حساسة تسجل في Audit Log.

لا تكتب تقرير وردي عام. اكتب تقرير حقيقي بالأدلة واذكر الملفات والجداول والاختبارات والنتائج.
